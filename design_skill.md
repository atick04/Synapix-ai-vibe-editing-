# Design Skill — Руководство по моушн-графике и UI-компонентам для AI Агента

Этот документ представляет собой полную спецификацию по моушн-дизайну и генерации интерактивной графики в системе **VibeEdit AI**. Руководство ориентировано на ИИ-агента, отвечающего за построение визуального ряда (сцен инфографики и интерфейсов), и содержит точные правила верстки, цветовые гаммы, шаблоны композиции, параметры анимации и структуры данных.

---

## 1. Философия дизайна: Стиль Apple Style & Минимализм

Основной визуальный стиль по умолчанию — **Apple Style** (премиальный минимализм с эффектом матового стекла / glassmorphism).

### Основные принципы:
1. **Эффект стекла (Glassmorphism):** Полупрозрачные подложки, тонкие светлые границы для создания глубины и мягкие размытые тени/свечения.
2. **Контрастность и Читаемость:** Текст на темных карточках должен быть максимально светлым (`#F5F7FA`), на светлых — контрастно темным (`#1C1C1E`). Критик дизайна автоматически контролирует это соотношение.
3. **Мягкие скругления:** Радиус скругления углов карточек составляет от `8px` (для мелких инпутов) до скругления половины высоты (`radius = height / 2`) для кнопок, навигационных панелей и вкладок.
4. **Микро-анимации:** Каждый элемент появляется с небольшим сдвигом по времени (эффект каскада / stagger) с плавными кривыми ускорения (easing).

## 2. Цветовые палитры и Шрифты

При генерации сцены в `style_profile` или стилях сущностей используются следующие палитры:

### Палитры цветов (`mood`):
*   **apple-dark** (Темная премиум тема):
    *   Фон карточек: `rgba(20, 20, 25, 0.65)`
    *   Граница: `rgba(255, 255, 255, 0.15)`
    *   Тень/Свечение: `rgba(255, 255, 255, 0.04)`
    *   Цвет текста: `#F5F7FA`
    *   Акцентный цвет: `#0A84FF` (Apple iOS Blue)
*   **apple-light** (Светлая премиум тема):
    *   Фон карточек: `rgba(255, 255, 255, 0.72)`
    *   Граница: `rgba(0, 0, 0, 0.08)`
    *   Тень/Свечение: `rgba(0, 0, 0, 0.02)`
    *   Цвет текста: `#1C1C1E`
    *   Акцентный цвет: `#007AFF` (Apple macOS Blue)
*   **cyberpunk** (Яркая технологичная тема):
    *   Фон карточек: `rgba(10, 10, 14, 0.80)`
    *   Граница: `rgba(0, 229, 255, 0.35)` (Neon cyan)
    *   Тень/Свечение: `rgba(0, 229, 255, 0.18)`
    *   Цвет текста: `#FFFFFF`
    *   Акцентный цвет: `#FF007F` (Neon pink)
*   **cozy-lofi** (Теплые органические тона):
    *   Фон карточек: `rgba(252, 248, 242, 0.88)`
    *   Граница: `rgba(140, 98, 57, 0.15)`
    *   Тень/Свечение: `rgba(140, 98, 57, 0.02)`
    *   Цвет текста: `#3E2723` (Deep brown)
    *   Акцентный цвет: `#D97706` (Terracotta amber)

### Правила типографики:
*   Используются **только кириллические шрифты** для корректного отображения русскоязычного текста:
    *   `Inter, sans-serif` (чистый, современный)
    *   `Montserrat, sans-serif` (универсальный, акцентный)
    *   `Rubik, sans-serif` (скругленный, дружелюбный)
    *   `Manrope, sans-serif` (геометричный, премиальный)
    *   `Unbounded, sans-serif` (сверхжирный, для кричащих заголовков)
    *   `Comfortaa, sans-serif` (мягкий, округлый)
    *   `JetBrains Mono, monospace` (код, аналитика)
    *   `Playfair Display, serif` (драматичные заголовки)
    *   `Marck Script, cursive` (изящный кириллический рукописный шрифт для комбинированных премиум-титров)

### Трендовый стиль Cursive Pairing (Aesthetic Cursive):
*   **Идея:** Комбинация строгого жирного шрифта без засечек (`Inter Bold` или `Montserrat Bold`) с элегантным рукописным шрифтом (`Marck Script`).
*   **Рецепт использования:**
    *   Базовый шрифт: `Inter_24pt-Bold`, цвет `#F5F5F7`
    *   Акцентный/парный шрифт: `Marck Script` (цвет `#FFFFFF` или пастельный акцент, курсив, размер увеличен на 1.3x)
    *   Регистр: `Sentence_Case` (для сохранения красоты прописных букв курсива)
    *   Настройки: `use_shadow: true`, `shadow_blur: 20`
    *   Пример word_styles: `'[[{"font": "Marck Script", "italic": true, "size": 1.3}, {"font": "Inter_24pt-Bold", "bold": true}]]'`

## 3. Границы безопасных зон (Safe Areas) и Пропорции

Графика должна адаптироваться под формат кадра. Координаты задаются в процентах от размера кадра (от `0` до `100`).

### Вертикальный формат (9:16, 1080x1920):
*   **Safe Area X:** `5.0% — 95.0%` (отступы слева и справа по 5%)
*   **Safe Area Y:** `15.0% — 76.0%`
    *   *Важно:* Верхняя зона (0-15%) резервируется под лицо спикера или шапку. Нижняя зона (76-100%) резервируется под кинетические субтитры. Графика не должна заходить в эти зоны!

### Горизонтальный формат (16:9, 1920x1080):
*   **Safe Area X:** `5.0% — 95.0%`
*   **Safe Area Y:** `10.0% — 90.0%`

## 4. Композиционные шаблоны (Auto-Layout Templates)

Если у элементов не указаны явные координаты `x` и `y`, графический движок расставляет их автоматически по одному из шаблонов:

1.  **`comparison`** (Сравнение):
    *   Разделяет экран на 2 симметричные колонки (сторона к стороне).
    *   Используется для сравнения параметров "Было/Стало", "До/После" или двух концепций.
2.  **`vertical_stack`** / **`list`** (Вертикальная стопка):
    *   Располагает карточки одна под другой сверху вниз с равными промежутками.
    *   Идеально для списков, шагов инструкции, чек-листов.
3.  **`timeline`** (Временная шкала):
    *   Расставляет элементы горизонтально слева направо.
    *   Используется для демонстрации этапов процесса, истории развития проекта.
4.  **`concept_explainer`** / **`mindmap`** (Связи/Интеллект-карта):
    *   Помещает одну главную родительскую карточку по центру сверху.
    *   Дочерние карточки выстраиваются в ряд ниже и соединяются стрелками со стартовым элементом.

## 5. Интерактивные UI-компоненты (Типы сущностей)

Сущности (`entities`) в сцене могут быть как стандартными текстовыми блоками, так и функциональными элементами интерфейса:

### 1. Headline (`type: "headline"`)
*   Крупный заголовок без фоновой рамки.
*   *Особенность:* Рендерится со специальной контрастной обводкой (stroke) и тенью для идеальной читаемости поверх любого видео-материала.

### 2. Navbar (`type: "navbar"`)
*   Премиальная панель навигации в стиле iOS.
*   **Параметры:**
    *   `text`: Название/логотип (слева).
    *   `items`: Список ссылок по центру (массив строк, например `["Главная", "Услуги", "Цены"]`).
    *   `action_text`: Текст кнопки действия справа (например, `“Войти”` или `“Начать”`).

### 3. Input Field (`type: "input_field"`)
*   Поле ввода данных.
*   **Параметры:**
    *   `label`: Текст надписи сверху инпута (заглавными буквами).
    *   `text`: Значение внутри инпута (если отсутствует — отображается как полупрозрачный placeholder `Enter text...`).
    *   `icon`: Идентификатор иконки (эмодзи), например `rocket`, `search`, `mail` (отображается слева в поле ввода).

### 4. Button (`type: "button"`)
*   Кнопка действия.
*   **Параметры:**
    *   `text`: Текст внутри кнопки.
    *   `icon`: Опциональная иконка перед текстом.
    *   `style_variant`: `"filled"` (залитая акцентным цветом кнопка) или `"outline"` (прозрачная кнопка с акцентной рамкой).

### 5. Tab Bar (`type: "tab_bar"`)
*   Переключатель вкладок/режимов.
*   **Параметры:**
    *   `items`: Список названий вкладок.
    *   `active_index`: Индекс активной вкладки (выделяется скругленной белой/акцентной подложкой).

### 6. Loading Bar (`type: "loading_bar"`)
*   Анимированная полоса прогресса. Заполняется синхронно с течением времени сцены от 0% до 100%.

### 7. Code Terminal Block (`type: "code_block"`)
*   Блок терминала / редактора кода в стиле macOS.
*   **Параметры:**
    *   `text`: Текст (код), разделенный символами `\n` или `\\n` для переноса строк.
    *   `label` или `title`: Заголовок/имя файла в шапке терминала (по умолчанию `terminal.sh`).

### 8. Metric/Stat Card (`type: "metric_card"`)
*   Карточка метрики с поддержкой динамического тренда/изменения.
*   **Параметры:**
    *   `value` или `number` или `text`: Главное крупное числовое значение (например, `"$12,450"`, `"98.4%"`).
    *   `label` или `sublabel`: Подпись снизу под числом (например, `"Месячный доход"`, `"Конверсия"`).
    *   `trend`: Строка тренда (например, `"+12%"` или `"-4%"`). Если передано, справа от числа рисуется красивая зеленая/красная плашка с процентом.

### 9. Circular Progress Indicator (`type: "circular_progress"`)
*   Круговой индикатор выполнения процесса (радиальный прогресс-бар).
*   **Параметры:**
    *   `progress`: Опциональный процент выполнения (число от `0` до `100`). Если не передан, прогресс плавно заполняется со временем сцены.
    *   `text` или `label`: Подпись снизу под круговым элементом.

### 10. Audio Waveform (`type: "audio_waveform"`)
*   Анимированный осциллятор звуковой волны (12 колеблющихся столбцов).
*   **Параметры:**
    *   `color` (внутри `styles` или `color_accent` в `style_profile`): Цвет столбцов. Высота столбцов автоматически колеблется по синусоидальной траектории во времени.

### 11. Sparkline Chart (`type: "sparkline"`)
*   Минималистичный векторный линейный график с эффектом постепенной прорисовки и светящимся маркером.
*   **Параметры:**
    *   `data`: Массив чисел от `0` до `100` (например, `[20, 45, 30, 80, 60, 95]`), передаваемый напрямую в виде массива или в виде JSON-строки. Задает высоты контрольных точек.
    *   `text` или `label`: Заголовок/подпись, отображаемая над графиком по центру.

### 12. Toggle Card (`type: "toggle_card"`)
*   Карточка настроек в стиле iOS с плавно переключающимся тумблером.
*   **Параметры:**
    *   `text` или `label`: Текст настройки, отображаемый слева (например, `"Включить уведомления"`).
    *   Анимация бегунка и изменение фонового цвета (с серого на зеленый) привязаны к `easeProgress` во время воспроизведения.

### 13. Profile Card (`type: "profile_card"`)
*   Премиальная карточка пользователя с круглым градиентным аватаром, именем, бейджем верификации (синяя галочка) и текстом отзыва/цитаты.
*   **Параметры:**
    *   `initials`: Инициалы для отображения на аватаре (например, `"JD"`). Если не указаны, берется первые 2 символа от `label`.
    *   `name` или `username`: Отображаемое имя пользователя (например, `"@iam_developer"`).
    *   `text`: Текст отзыва или комментария, который автоматически переносится по словам в пределах карточки.

### Свойства статуса (Badge)
Любая сущность (`entity`), независимо от ее типа, поддерживает два дополнительных параметра:
*   `badge`: Строка статуса (например, `"NEW"`, `"HOT"`, `"LIVE"`), которая рендерится в виде маленькой капсульной плашки на верхнем правом углу карточки.
*   `badge_color`: Цвет плашки статуса (например, `#FF3B30` или `#34C759`). По умолчанию используется фиолетовый акцент `#5856D6`.

## 6. Анимация (Motion & Relations)

Каждый элемент поддерживает детальные настройки появления и движения:

*   **Типы анимации (`animation.type`):**
    *   `fade` — плавное проявление прозрачности.
    *   `pop` — масштабирование от 50% до 100% с эффектом прыжка.
    *   `slide_in` — выезд слева/снизу со смещением.
*   **Кривые сглаживания (`animation.easing`):**
    *   `smooth` — стандартная плавная bezier-кривая.
    *   `bounce` — пружинящая кривая (идеально для `pop` анимации карточек).
    *   `linear` — постоянная скорость (для шкал прогресса `loading_bar`).
*   **Связи (`relations`):**
    *   Задают направленные стрелки между сущностями (`from` -> `to`). Стрелки анимируются с эффектом прогрессивного рисования линии (draw-in) сразу после того, как исходный элемент появился на экране.

## 8. Совмещение субтитров и графики: Правила для текстовых хуков и плашек

Для достижения максимального удержания внимания (Retention) ключевые смысловые фразы, хуки начала видео и важные цифры должны переноситься из субтитров в графический вид:

1. **Крючки внимания (Hooks):**
   * Если спикер в начале видео произносит цепляющую фразу (например: *"И это обучение стоит 1000 сомов"*), создайте графическую сцену на первые 2-4 секунды.
   * Вместо стандартного субтитра используйте карточки типа `stat_card` с премиальной стеклянной подложкой или крупный заголовок `headline` с эффектом пружинящего появления `pop` или `slide_in`.
   * Пример JSON для хука:
     ```json
     {
       "start_time": 0.0,
       "duration": 3.0,
       "scene_template": "list",
       "mood": "apple-dark",
       "entities": [
         {
           "id": "hook_panel",
           "type": "stat_card",
           "text": "И ЭТО ОБУЧЕНИЕ\nСТОИТ 1000 СОМОВ",
           "x": 50.0,
           "y": 45.0,
           "width": 80.0,
           "height": 18.0,
           "animation": {
             "type": "pop",
             "easing": "bounce",
             "duration": 0.6
           }
         }
       ]
     }
     ```

2. **Согласование стилей и шрифтов:**
   * Графический агент обязан настроить шрифты субтитров через инструмент `build_kinetic_typography` так, чтобы они соответствовали выбранному `mood` сцены:
     * Для тем `apple-dark` / `apple-light` -> субтитры со шрифтом `Montserrat-ExtraBold` или `Inter_24pt-Bold`.
     * Для темы `cyberpunk` -> субтитры со шрифтом `Montserrat-ExtraBold` или `Unbounded-Bold`, яркий акцентный цвет (например, `#FF007F` или `#00E5FF`).
     * Для темы `cozy-lofi` -> субтитры со шрифтом `Comfortaa-Bold` или `Rubik-Bold` с мягкими цветами.

## 9. Руководство по моушн-дизайну от LottieFiles (Motion Design Principles)

Графический агент должен проектировать анимации появления и взаимодействия элементов, следуя стандартам моушн-дизайна LottieFiles.

### 1. Архетипы анимации (Motion Personality):
*   **Premium** (Используется для тем `apple-dark` / `apple-light`):
    *   Длительность: `0.35 — 0.5` сек.
    *   Сглаживание (`easing`): `smooth` (плавная кривая без отскоков).
    *   Характер: сдержанный, дорогой, минималистичный.
*   **Playful** (Для блогов, развлекательного контента):
    *   Длительность: `0.2 — 0.35` сек.
    *   Сглаживание (`easing`): `bounce` (эффект пружинящего отскока / overshoot).
    *   Характер: фановый, дружелюбный, прыгающий.
*   **Corporate** (Для графиков, аналитики):
    *   Длительность: `0.2 — 0.4` сек.
    *   Сглаживание (`easing`): `ease-out` (быстрый старт и мягкое замедление).
    *   Характер: чистый, профессиональный, собранный.
*   **Energetic** (Для динамичных склеек, Shorts, техно-обзоров):
    *   Длительность: `0.1 — 0.25` сек.
    *   Сглаживание (`easing`): `ease-out-expo` (экспоненциальный выезд с максимальной скоростью в начале).

### 2. Направление сглаживания (Easing Rules):
*   **Входные анимации (Entrances)**: Должны быстро привлекать внимание и плавно замедляться. Используйте `ease-out`, `ease-out-back` (`bounce`) или `ease-out-expo`.
*   **Выходные анимации (Exits)**: Должны быстро покидать экран. Используйте `ease-in` (плавный старт, резкое ускорение при уходе).
*   **Постоянно присутствующие на экране (On-screen)**: Используйте `smooth` для плавности с обоих концов.

### 3. Правила хореографии и качественной сборки (Quality Checklist):
1.  **Запрет на линейные анимации для движения**: Линейная интерполяция (`linear`) разрешена только для шкал прогресса (`loading_bar`) или вращающихся спиннеров. Любое движение карточек/заголовков должно быть сглажено кривыми!
2.  **Слои движения (Motion Layers)**: Каждая сцена должна обладать глубиной:
    *   *Первичный слой (Primary)*: Появление основной карточки.
    *   *Вторичный слой (Secondary)*: Появление текста и иконок внутри карточки с небольшой stagger-задержкой (`0.05 - 0.1s`).
    *   *Фоновый слой (Ambient)*: Плавное дыхание или пульсация фонового свечения.
3.  **Правило 1/3 (Choreography)**:
    *   *По расстоянию*: Элемент не должен пролетать более 1/3 кадра по прямой без промежуточных ключевых кадров или сильного замедления.
    *   *По элементам*: При появлении более 3 элементов на экране, одновременно в активном движении должна находиться максимум 1/3 элементов (используйте каскад задержек `stagger` от `0.05` до `0.15` сек). Максимальный бюджет появления всей сцены — `0.5` сек.
